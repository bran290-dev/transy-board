# URL로 소설 추가

웹소설 사이트의 URL을 입력하여 소설을 추가하는 방법입니다. 연재 중인 소설을 추적하고 새 회차를 자동으로 가져올 수 있습니다.

---

## 지원 사이트

Transy는 파싱 룰을 통해 다양한 웹소설 사이트를 지원합니다.

### 기본 제공 파싱 룰
기본 제공 파싱 룰은 현재 지원되는 사이트가 없으며, 지원 사이트 목록은 지속적으로 업데이트 예정입니다.
### 커스텀 파싱 룰

지원되지 않는 사이트는 파싱 룰 생성기를 통해 직접 룰을 만들 수 있습니다.
→ [파싱 룰 생성](../advanced/parsing-rules.md) 참조

---

## 추가 단계

### 1단계: URL 탭 선택

라이브러리에서 **+** 버튼을 클릭한 후, **URL** 탭을 선택합니다.

[사진: URL 탭 선택 화면]

### 2단계: URL 입력

웹소설의 **목록 페이지 URL**을 입력합니다.

[사진: URL 입력 필드]

> **팁**: 개별 회차 URL이 아닌, 전체 회차 목록이 있는 페이지 URL을 입력하세요.

### 3단계: 불러오기

"소설 정보 가져오기" 버튼을 클릭하면 파싱이 시작됩니다.

[사진: 소설 정보 가져오기 버튼]

### 파싱 과정

1. URL에 맞는 파싱 룰 자동 탐색
2. 소설 메타데이터 추출 (제목, 작가, 설명)
3. 회차 목록 추출

[사진: 파싱 진행 중]

## 소설 정보 확인

파싱이 완료되면 추출된 정보가 표시됩니다.

[사진: 소설 정보 미리보기 화면]

### 표시 항목

| 항목 | 설명 |
|------|------|
| 제목 | 소설 제목 (편집 가능) |
| 작가 | 작가명 |
| 설명 | 소설 소개/줄거리 (편집 가능) |
| 회차 수 | 추출된 총 회차 수 |
| 출처 | 원본 사이트 URL |

### 정보 수정

추출된 제목이나 설명이 잘못된 경우 직접 수정할 수 있습니다.

[사진: 제목/설명 편집]

## 주의사항

### URL 형식

- **올바른 예**: `https://example.com/novel/12345` (목록 페이지)
- **잘못된 예**: `https://example.com/novel/12345/chapter/1` (개별 회차)

### 회차 순서

일부 사이트는 회차가 역순으로 표시될 수 있습니다. 파싱 룰에서 자동으로 처리되지만, 순서가 이상한 경우 문의해 주세요.

### 로그인 필요 사이트

로그인이 필요한 사이트도 **파싱 룰 생성기의 인증** 기능을 통해 지원됩니다.

#### 인증 지원 방식

1. **파싱 룰 생성** 시 인증이 필요한 사이트 감지
2. **WebView 창**이 열려 직접 로그인/CAPTCHA 입력 가능
3. 인증 완료 후 이후 접근에 사용

[사진: WebView 인증 화면]

> **참고**: 인증 정보는 앱 내에 안전하게 저장되며, 세션 만료 시 재인증이 필요할 수 있습니다.

→ [파싱 룰 생성](../advanced/parsing-rules.md) 가이드에서 자세한 내용 확인

## 문제 해결

### "파싱에 실패했습니다"
- 사이트가 정상적으로 접근 가능한지 확인
- 파싱 룰이 최신인지 확인
- 사이트 구조가 변경되었을 수 있음

### "회차를 찾을 수 없습니다"
- 소설에 등록된 회차가 있는지 확인
- 목록 페이지 URL이 맞는지 확인

### 일부 회차만 추출됨
- 페이지네이션이 있는 경우 첫 페이지만 추출될 수 있음
- 파싱 룰 업데이트 필요

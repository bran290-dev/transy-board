# 컨텍스트 시스템 개요

Transy의 컨텍스트 시스템이 어떻게 작동하는지 설명합니다.

---

## 컨텍스트 시스템이란?

AI 번역 시 추가 정보를 제공하여 번역 품질을 향상시키는 시스템입니다.

[사진: 컨텍스트 시스템 개념도]

---

## 구성 요소

### 5가지 컨텍스트 유형

| 유형 | 설명 | 관리 방식 |
|------|------|----------|
| **캐릭터** | 등장인물 정보 | 자동+수동 |
| **용어** | 고유명사/전문용어 | 자동+수동 |
| **사건** | 주요 사건 기록 | 자동 |
| **대사** | 중요 대사 기록 | 자동 |
| **회차 요약** | 각 회차 줄거리 | 자동 |

---

## 작동 원리

### 번역 시 컨텍스트 주입

1. 번역 요청 시작
2. 관련 컨텍스트 수집
   - 현재 회차까지의 캐릭터
   - 등록된 용어
   - 최근 사건/대사
   - 이전 회차 요약
3. 번역 프롬프트에 컨텍스트 추가
4. AI가 컨텍스트를 참고하여 번역

[사진: 컨텍스트 주입 과정]

### 자동 추출

번역 과정에서 AI가 자동으로:
- 새 캐릭터 감지 및 등록
- 새 용어 감지 및 등록
- 주요 사건 기록
- 회차 요약 생성

## 컨텍스트 윈도우

### 윈도우란?

번역 시 참조할 이전 회차 범위입니다.

[사진: 컨텍스트 윈도우 설정]

### 설정

소설별 설정에서 조정:

| 설정 | 기본값 | 설명 |
|------|--------|------|
| 컨텍스트 윈도우 | 3회차 | 이전 N개 회차 참조 |

### 예시

현재 10회차 번역 시, 윈도우가 3이면:
- 7, 8, 9회차의 컨텍스트 참조
- 더 넓은 맥락 이해 가능

---

## 자동 보정

### 기능 설명

번역 후 등록된 캐릭터/용어로 자동 치환합니다.

### 예시

```
등록된 캐릭터: 田中 → 다나카

번역 결과: "타나카가 말했다"
자동 보정: "다나카가 말했다"
```

### 설정

소설별 설정에서:
- 캐릭터 자동 보정: ON/OFF
- 용어 자동 보정: ON/OFF

## 컨텍스트 버전 관리

### 버전이란?

컨텍스트가 변경되면 버전이 업데이트됩니다.

### 번역 캐시 무효화

컨텍스트 변경 시 관련 번역 캐시가 무효화될 수 있습니다.

[사진: 번역 캐시 무효화 다이얼로그]

### 옵션

| 옵션 | 설명 |
|------|------|
| 번역 유지 | 기존 번역 그대로 유지 |
| 일괄 치환 | 변경된 이름/용어만 치환 |
| 번역 삭제 | 번역 삭제 후 재번역 필요 |

## 회차 필터

특정 회차까지의 컨텍스트만 볼 수 있습니다.

[사진: 회차 필터]

### 용도

- 스포일러 방지
- 특정 시점의 컨텍스트 확인
- 캐릭터 첫 등장 확인

### 사용 방법

1. 컨텍스트 관리 화면 상단
2. 회차 필터 선택
3. 해당 회차까지의 정보만 표시

컨텍스트 내에서 검색할 수 있습니다.

## 효과적인 사용 팁

### 1. 주요 캐릭터 미리 등록

소설 시작 전에 주요 캐릭터를 등록하면:
- 첫 회차부터 일관된 번역
- 자동 추출보다 정확

### 2. 중요 용어 수동 관리

자동 추출이 부정확할 수 있는 용어:
- 작품 고유 용어
- 특수한 지명
- 능력/스킬명

### 3. 정기적인 검토

10~20회차마다:
- 잘못된 캐릭터 정보 수정
- 중복 항목 정리
- 설명 보완

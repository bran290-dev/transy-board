# 사건/대사 관리

소설의 주요 사건과 중요 대사를 추적합니다.

---

## 사건 탭

주요 사건을 기록하여 스토리 맥락을 유지합니다.

[사진: 사건 탭 화면]

---

## 사건이란?

스토리의 중요한 전환점이나 이벤트입니다.

### 예시

- 주인공의 각성
- 중요 캐릭터와의 만남
- 전투의 결과
- 비밀 발견

---

## 사건 목록

### 표시 정보

| 항목 | 설명 |
|------|------|
| 제목 | 사건 제목 |
| 설명 | 사건 상세 설명 |
| 등장 캐릭터 | 관련 캐릭터 |
| 발생 회차 | 사건이 일어난 회차 |

[사진: 사건 항목 상세]

---

## 자동 추출

번역 과정에서 AI가 주요 사건을 자동으로 감지합니다.

### 감지 기준

- 스토리의 중요 전환점
- 캐릭터 상태 변화
- 새로운 정보 발견
- 갈등/해결 상황

### 추출 결과

```
제목: 주인공의 마법 각성
설명: 위기 상황에서 처음으로 마법을 사용하게 됨
등장 캐릭터: 김철수, 마법사
발생 회차: 15회차
```

---

### 삭제 방법

1. 사건 항목에서 삭제 버튼
2. 확인 후 삭제

## 대사란?

### 예시

- 핵심 정보가 드러나는 대사
- 캐릭터 관계를 보여주는 대사
- 복선이 되는 대사

## 자동 추출

AI가 중요한 대사를 자동으로 감지합니다.

### 감지 기준

- 스토리에 영향을 주는 대사
- 캐릭터 관계를 나타내는 대사
- 정보가 포함된 대사

---

## 대사 북마크

중요한 대사에 북마크를 표시할 수 있습니다.

[사진: 대사 북마크]

### 용도

- 나중에 참조할 대사 표시
- 중요 대사 우선 표시

### 설정 방법

1. 대사 항목의 북마크 아이콘 클릭
2. 북마크 활성화/비활성화

---

## 대사 삭제

불필요한 대사를 삭제할 수 있습니다.

### 삭제 방법

1. 대사 항목에서 삭제 버튼
2. 확인 후 삭제

---

## 사건/대사의 역할

### 번역 품질 향상

- 이전 사건을 참고하여 맥락 유지
- 관련 대사로 상황 이해
- 캐릭터 행동 동기 파악

### 예시

```
사건: "철수가 마법을 각성함" (15회차)

20회차 번역 시:
→ AI가 철수의 마법 각성을 알고 있음
→ 관련 내용을 적절히 번역
```

## 활용 팁

### 1. 정기적 검토

자동 추출된 사건/대사 중:
- 중요하지 않은 것 삭제
- 중요한 것 북마크

### 2. 스포일러 주의

뒷 회차의 사건이 번역에 영향을 주지 않도록:
- 컨텍스트 윈도우 적절히 설정
- 필요시 회차 필터 사용

### 3. 컨텍스트 크기

사건/대사가 너무 많으면:
- 번역 시간 증가
- 불필요한 정보 삭제 권장

- 수정 미지원 (삭제/북마크만 가능)
